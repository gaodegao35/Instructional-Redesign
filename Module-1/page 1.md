Module 1 Getting Organized
Learning Objective: 
Understand why organizing things is necessary
Differentiate between what you have (things) and what you owe (promises)
Connect lists to accounting terms (assets and liabilities)
Decide what to include into each list

Context: your and the bike shop you own

Page 1: the need to organize
What you’ll discover:
Why you need things organized
How to get things organized: 
Write them down
Group them together
Name each group and keep a structure
Fit new things into the structure

Scenario: You run a bicycle repair shop. It has been six months. Tools are scattered, parts are stacked in corners, and you have been keeping most things in your head. It isn’t really working anymore. You want to get more systematic about how you run this thing.

Flip cards
Three things happened this week. None of them went smoothly.
A customer walks in with a broken derailleur. “Do you have a 24mm wrench? I left mine at home.” You think you do, somewhere in that toolbox under the bench, or maybe in the drawer. You start digging.
Tonight you unwrapped a fork crown race tool you bought last week and found another one already in the drawer, still in its package. Third time this month you’ve reordered something you already had.
A fire two blocks over last night gets you thinking: you should have insurance, and you don’t really know what’s at risk. The repair stand, the parts, all the tools, it’s a lot.

Call out: Three different moments, the same realization: To get things organized, it’s time to write things down and create a list.

Scenario: You decide to look into insuring the things you use in the shop. You call Maria — the agent your hardware-store friend uses. “Great. Eventually I’ll need a detailed list — every wrench, every tube, every bracket, with a value for each. But to draft a policy I just need a summary. Group your stuff in a way that makes sense, give me a rough value for each group, and we’ll work from that.”

You start sorting things into different groups.
Activity 1: Drag and Drop
[create a drag and drop activity. Create 12 items that belong to a bicycle shop. The items should belong to 4 different categories. Learners need to drag each item into a group. Then after grouping, learners need to name each group. ]

Callout: You draw four lines to separate things. In each group, you have a reason to put items together, and the insurance agent will work with this structure.

Scenario: Friday morning. A delivery arrives — a new box of brake pads. Where does it go?
Activity 2: MCQ
[create a MCQ activity. Ask learners to select which group this new item should belong to. The name of each group is the name learners created in last activity. After learners select the correct choice, show the following text as feedback “You found the group without much thought — because the bin made sense. The structure you built can absorb new things as they arrive. That’s what categories do. You’ll be glad you have them.”]

Callout: You took 12 separate items and built four lines you can think with. That’s the move from raw detail to manageable structure. The same move applies to every list you’ll ever maintain — and you’re about to start maintaining yours.
